# 神山式トークスクリプト
**用途**: 新人〜中堅営業向け再現可能な台本
**ベース**: 神山の成約顧客18名の通話分析から抽出

---

## 【新人用】基本トークスクリプト

### STEP 1: 初回通話（目標: 次回予約を取る）

#### 開始〜3分: 掴み

```
「お忙しい中ありがとうございます。
 ○○（前担当）から引き継ぎました、△△と申します。
 
 ○○からお客様の状況はしっかり共有いただいておりますので、
 改めてご挨拶と、今後のサポート体制についてお話しさせてください。
 
 まず、何かご不安な点やお困りのことはありますか？」
```

**ポイント:**
- 最初の3分は**聞く**。話さない
- 顧客の不安・不満を吐き出させる
- 「はいはいはい」ではなく「なるほど、それは心配ですよね」と共感

#### 3分〜10分: 信頼構築（詐欺差別化3点セット）

```
「率直にお話しします。
 この業界、残念ながら不誠実な業者が多いのは事実です。
 
 弊社が他と違う点を3つだけお伝えさせてください。
 
 1つ目: 入金先はお客様ご自身の正規口座です。
    弊社にお金を預けるのではありません。ご自身で管理できます。
 
 2つ目: 報酬は成功報酬です。
    利益が出なければ弊社の報酬はゼロです。
    つまり、お客様が儲からなければ弊社も儲かりません。
 
 3つ目: 契約書は弁護士監修です。
    全て書面に残ります。口約束は一切ありません。
 
 まず実力を見ていただいて、
 信頼できると思ったら次に進む。
 それで全く問題ありません。」
```

**ポイント:**
- この3点は**必ず初回で伝える**。省略不可
- 顧客が詐欺被害者の場合、「見極め方」として教える形で
- 「報酬を受け取るタイミングが全て」が判断基準

#### 10分〜: 状況ヒアリング

```
「いくつか確認させてください。
 
 ・現在、仮想通貨の保有はありますか？
 ・これまでの投資経験はどのようなものがありますか？
 ・何か目標とされている金額はありますか？
 
 【顧客の回答を聞いた後】
 
 ありがとうございます。
 ○○様の状況に合わせて、最適なプランをご提案させていただきます。
 
 一度に全部ではなく、まずは小さく始めて、
 弊社の実力を見ていただくのが一番だと思います。
 
 来週の○曜日、○時あたりはいかがですか？
 30分もあれば十分です。」
```

**ポイント:**
- **必ず次回予約を取って終わる**
- 初回で商品を売らない。売るのは「次の約束」だけ
- 目標金額を聞くのは今後の提案設計のため

---

### STEP 2: 2-3回目（目標: 小額テスト投資の実行）

#### 信頼の深化

```
「前回お話しした内容、いかがでしたか？
 何か気になったことや、調べてみたことはありますか？
 
 【顧客の質問・不安に答える】
 
 今日は、実際の運用について少しお話しさせてください。
 
 ○○様の目標は△△万円でしたね。
 
 結論から言うと、十分現実的な数字です。
 理由を3つお伝えします。
 
 1つ目: [具体的な市場データ]
 2つ目: [過去の実績データ]
 3つ目: [今のタイミングの優位性]
 
 まずは痛手にならない金額で、
 弊社の実力を確かめていただけますか。
 ○○万円程度で始めて、1-2ヶ月で結果をお見せします。」
```

**ポイント:**
- 「結論→理由→具体例→顧客の利益」の順
- 金額は顧客の経済状況に合わせる（痛手にならない額）
- 「実力を見る」＝リスクが低い、お試しという心理

#### 操作サポート（画面共有）

```
「では一緒にやっていきましょう。
 画面共有させていただいてもよろしいですか？
 
 【AnyDeskで接続】
 
 まず右上の○○を押してください。
 はい、完璧です。
 
 次に…
 
 【1ステップずつ、確認しながら進める】
 
 これで完了です。お疲れ様でした。
 何か分からないことがあれば、いつでも連絡ください。」
```

**ポイント:**
- 一度に一つの指示。複数同時に出さない
- 「完璧です」「素晴らしいです」と褒める
- 完了後に必ず「何かあればいつでも」

---

### STEP 3: 中盤（目標: 定期報告で信頼強化）

#### 週次報告テンプレート

```
「○○様、お疲れ様です。△△です。
 今週の状況をご報告します。
 
 【数字で報告】
 ・現在の評価額: ○○万円（先週比+○○万円）
 ・利益率: ○○%
 
 順調に推移しています。
 来週も引き続き見ていきますので、ご安心ください。
 
 何かご質問があればいつでもどうぞ。」
```

**ポイント:**
- **必ず数字で報告する**。「順調です」だけは禁止
- 下がっている時こそ速く報告する。隠さない
- 「来週も見ています」＝放置していない安心感

#### 暴落時の対応

```
「○○様、お疲れ様です。△△です。
 市場の状況について、すぐにお伝えしたくてご連絡しました。
 
 現在、一時的に価格が下がっています。
 ただ、これは市場全体の動きで、
 ○○（銘柄名）に固有の問題ではありません。
 
 過去にも同様の調整は何度もありましたし、
 その後は回復しています。
 
 今のところ、保有を継続するのが最善だと判断しています。
 理由は…
 
 もし不安なことがあれば、いつでもお電話ください。
 ○○様のお考えを最優先にしますので。」
```

**ポイント:**
- 暴落時に連絡しない＝信頼崩壊
- 「会社の利益より、お客様の安心が優先」と明言する
- 売却の選択肢も残す（「ご判断を最優先にします」）

---

### STEP 4: クロージング（目標: 追加投資の獲得）

#### 段階的引き上げ

```
「○○様、前回のNFTで○○万円の利益が出ましたね。
 おめでとうございます。
 
 次のステップとして、2つの選択肢をご用意しました。
 
 A: ○○万円で [商品A]（利益見込み: 2-3倍、期間: 1-2ヶ月）
 B: ○○万円で [商品B]（利益見込み: 3-5倍、期間: 3-6ヶ月）
 
 私の推奨はAです。
 理由は[具体的な理由]。
 
 もちろん、無理にとは言いません。
 ○○様のペースで判断していただければ結構です。」
```

**ポイント:**
- 成功体験の直後に次の提案を出す
- 必ず選択肢を2つ以上出す
- 推奨を明確にする（「どうしますか？」は禁止）
- 断られたら即引く。押さない

#### 希少性の演出

```
「実は今回の案件、タイミングが重要でして、
 今月中に動かないと次のチャンスは[時期]になります。
 
 他のお客様も今このタイミングで動いています。
 
 ただ、焦って判断する必要はありません。
 ○○様が納得された上で進めるのが一番です。
 
 明日までにお考えいただいて、
 お返事いただけますか？」
```

## 【経験者用】フレームワーク

経験者は台本通りではなく、以下のフレームワークを状況に応じて使い分ける。

### フレームワーク1: 顧客タイプ判定（初回で判定）

| 質問 | 判定 |
|---|---|
| 過去に詐欺被害がある？ | → 詐欺差別化を最優先 |
| 投資経験はある？ | → 論理・データ重視で対応 |
| IT操作に不安がある？ | → 画面共有サポートを厚く |
| 感情的に揺れやすい？ | → 傾聴・定期フォローを厚く |
| 前業者に不満がある？ | → 「味方」宣言→解約支援 |
| 即結果を求める？ | → 緊急性・希少性を前面に |

### フレームワーク2: 説明の型

```
複雑な話 → 「2つに分けましょう」
結論を出す → 「結論→理由→具体例→顧客の利益」
リスクを伝える → 「リスクはこう。ただし緩和策はこう」
比較する → 「Aのメリット・リスク / Bのメリット・リスク。推奨はA」
```

### フレームワーク3: クロージング判断

```
顧客が前向き → 「決めちゃっていいんじゃないですか」（軽く押す）
顧客が迷い中 → 「明日までにお考えください」（時間を与える）
顧客が消極的 → 「無理にとは言いません。次の機会に」（即引く）
顧客が反対  → 「おっしゃる通りです」（肯定して撤退。関係維持優先）
```

### フレームワーク4: 通話構造（1回あたり）

| 時間 | 内容 |
|---|---|
| 0-3分 | 挨拶・状況確認・アイスブレイク |
| 3-5分 | 前回の続き・進捗報告 |
| 5-20分 | 本題（説明・提案・質問対応） |
| 20-25分 | まとめ・次のアクション確認 |
| 最後1分 | **次回予約確定**（省略不可） |

## チェックリスト

### 初回通話チェック
- [ ] 顧客の不安・不満を聞き出したか
- [ ] 詐欺差別化3点を伝えたか
- [ ] 目標金額を聞いたか
- [ ] 次回予約を取ったか
- [ ] 初回で商品を売っていないか（売ったらNG）

### 毎回チェック
- [ ] 数字で報告したか（「順調です」は禁止）
- [ ] 次回予約を確定したか
- [ ] 顧客の名前を正しく呼んだか
- [ ] 推奨を明確にしたか（「どうしますか？」は禁止）

### クロージングチェック
- [ ] 選択肢を2つ以上出したか
- [ ] 推奨を明確にしたか
- [ ] 断られた時に即引いたか
- [ ] 「無理にとは言わない」を伝えたか
