# PJ-008 Phase④: X（Twitter）自動運用 設計書

## コンセプト
Xは送客ハブ。Threads/ブログ/noteへの流入を増やす装置。
X単体での収益化は狙わない。拡散力を使う。

## アカウント
- X ID: @ken_fit_science（Threadsと統一）
- プロフィール: 🔬筋トレ×科学｜論文ベースで本当に効くことだけ発信

## 投稿戦略

### コンテンツ（1日3ツイート）
1. **朝（8:00）**: Threadsの投稿を短縮版（140字以内）で投稿
2. **昼（12:00）**: ブログ記事のリンク付きツイート
3. **夜（20:00）**: 参加型（質問・アンケート）

### 導線設計
```
Xツイート
  → プロフィールリンク（ブログ or note）
  → ブログ（SEOアフィリ）/ note（有料記事）
```

### ハッシュタグ戦略
- #筋トレ #プロテイン #ダイエット #ボディメイク #フィットネス
- トレンドタグに便乗（季節イベント等）

## cron
| タスク | 頻度 | 実行者 |
|--------|------|--------|
| 朝ツイート | 毎日8:00 | Sonnet |
| 昼ツイート（ブログリンク） | 毎日12:00 | Sonnet |
| 夜ツイート（参加型） | 毎日20:00 | Sonnet |
| フォロー・いいね活動 | 毎日15:00 | Sonnet |

## 課題
- Xアカウント作成にはメールor電話番号が必要
- API不要（ブラウザ操作でOK）
- 凍結リスクがThreadsより高い → 投稿頻度を控えめに
